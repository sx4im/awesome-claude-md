## Architecture Rules

- **Define once, generate everywhere.** Define styles in TypeScript, Panda generates CSS at build time.
- **Use patterns for atomic CSS.** Panda generates atomic classes. Use `css()` function for styling.
- **Recipes for component variants.** Define component styles with `defineRecipe()` for variants (sizes, colors, states).
- **Tokens for design system values.** Colors, spacing, fonts defined in `tokens.ts` become type-safe CSS variables.

## Coding Conventions

- Import `css` from `styled-system/css`: `import { css } from '../../styled-system/css'`.
- Apply styles: `className={css({ bg: 'blue.500', p: '4' })}`.
- Use recipes: `const button = cva({ ... })` from `styled-system/css`.
- Style props use theme token references: `p: '4'` not `p: '16px'`.
- Conditional styles with `css.raw()` or object conditions: `css({ color: { base: 'red', md: 'blue' } })`.

## Library Preferences

- **Tokens:** Define in `panda.config.ts` or separate files. Panda generates type-safe references.
- **Recipes:** Use for any component with variants. Button, Badge, Card all deserve recipes.
- **Patterns:** Use built-in patterns (flex, grid, stack, container) for layouts.
- **Utilities:** Extend with custom utilities for project-specific needs.

## File Naming

- Panda config: `panda.config.ts` at project root.
- Token definitions: `tokens.ts` in panda directory.
- Recipes: `[component].ts` in `panda/recipes/`.

## NEVER DO THIS

1. **Never edit `styled-system/` directly.** It's generated. Modify `panda.config.ts` or source files.
2. **Never use raw values in `css()` calls.** Always reference tokens: `bg: 'red.500'` not `bg: '#ef4444'`.
3. **Never forget to run `prepare` script.** Panda needs to generate `styled-system` before development.
4. **Never skip token definitions.** Panda's power comes from tokens. Define your design system properly.
5. **Never mix Panda with Tailwind in the same project.** Both generate CSS, leading to conflicts.
6. **Never use `!important` in Panda styles.** Use specificity or the cascade correctly.
7. **Never ignore the `outdir` configuration.** Ensure `styled-system` is gitignored but accessible for imports.
